# BudgetModel

Token budget math and content density rules for CLAUDE.md files generated by ContextLayer.

---

## The Governing Principle: Context Window > Completeness

Every token in a CLAUDE.md file displaces working space in the agent's context window. Missing information causes a follow-up question. Wrong or stale information causes silent failures — the agent acts confidently on bad context.

**Optimize for agent accuracy, not for coverage.** If five conventions exist but four are inferable from the code, include only the one that isn't. The agent can read files on demand. It cannot un-read noise that polluted its context.

> Every entry must earn its place. The default for any line is deletion. Content survives only by proving an agent would fail without it.

---

## Content Density Rules

### Signal Tests (apply to every candidate entry)

Every CLAUDE.md entry must pass ALL of these tests to survive:

| Test | Question | Fails → |
|------|----------|---------|
| **Falsifiability** | If removed, would agent behavior degrade? | Remove it |
| **Prediction** | Would an agent infer this from the code alone? | Remove it — the code is the source of truth |
| **One New Fact** | Does this duplicate information from another bullet? | Keep the more specific version, delete the other |
| **State Once** | Does this appear in both root and subdirectory CLAUDE.md? | Keep it in the more specific (subdirectory) file only |
| **Project-Specificity** | Is this true of most projects, or just this one? | If true of most → remove. Only project-specific content survives |

### Entry Format Constraints

| Rule | Limit | Rationale |
|------|-------|-----------|
| **Words per bullet** | **15 max** | Forces precision. "Use kebab-case for all filenames" not "All files in this project should be named using kebab-case naming convention" |
| **Voice** | Active imperative | "Use kebab-case" not "Files should be named using kebab-case" |
| **Prose paragraphs** | **Banned** | Tables and bullets only. Prose wastes tokens on connective tissue the agent doesn't need |
| **Table cells** | **1 sentence max** | If a cell needs more, the table structure is wrong |
| **Sections per file** | **5-7 max** | More sections = more headers = more tokens on structure |

### Anti-Patterns for CLAUDE.md Content

| Pattern | Example | Why it fails |
|---------|---------|-------------|
| "This project uses X" | "This project uses TypeScript" | Agent infers from `.ts` files |
| General best practices | "Use meaningful variable names" | Agent already knows this |
| Hedging | "Consider using...", "It's recommended to..." | State the convention or don't. No hedging. |
| Version numbers (non-runtime) | "React 18.2.0" | High rot risk. Agent reads package.json. |
| "See also" references | "See: docs/architecture.md" | Agent won't follow them |
| Redundant with README | Getting started steps already in README | Agent can read README on demand |
| Unfalsifiable claims | "Good error handling is important" | True of every project. Zero information. |

## Napkin Math

CLAUDE.md budget is ~800–2,000 tokens out of ~200K context window. Each unnecessary token is lost working space, repeated every session. Target signal ratio > 0.8.

---

## Per-File Token Allocation

### Root CLAUDE.md (800–1500 tokens)

| Section | Token range | Format |
|---------|-------------|--------|
| Commands | 100–300 | Table: `cmd → purpose` |
| Conventions | 150–400 | Bullet list, project-specific only |
| File structure | 100–200 | Indented tree, 1-line annotations |
| Cross-boundary summaries | 100–400 | Inline: `name: owns (entry: path)` |
| Prohibitions | 50–150 | Bullet list |
| Pruning instruction | 50–120 | Embedded template from PruningInstruction.md |
| **Total** | **550–1,550** | Not all sections hit upper bounds simultaneously; trim to 1500 cap |

### Subdirectory CLAUDE.md (200–500 tokens)

| Section | Token range | Format |
|---------|-------------|--------|
| Local commands (if any) | 50–100 | Table or bullet |
| Local conventions | 75–200 | Bullet list |
| Key files | 75–150 | Table: `path → role` |
| Local constraints | 50–100 | Bullet list |
| Pruning instruction | 50–120 | Embedded template |
| **Total** | **300–650** | Not all sections hit upper bounds simultaneously; trim to 500 cap |

---

## Budget Enforcement Rules

1. **Count before writing.** Estimate token count of each section before including it.
2. **Commands win.** If budget is tight, commands survive. Everything else is negotiable.
3. **Trim conventions, not commands.** Cut the least-specific conventions first.
4. **Never exceed root budget for a subdirectory file.** Subdir files get roughly 1/3 the root budget ceiling.
5. **The pruning instruction is NOT optional.** It counts against budget but always stays.

---

## Budget Red Flags

If your generated CLAUDE.md is approaching 2000 tokens for root or 600 for subdir:
- Remove all "See also" references (they don't work anyway)
- Remove any prose paragraphs — convert to bullet lists
- Remove anything the agent could infer from the file tree alone
- Remove general best practices that any competent developer would apply
- Remove version numbers that aren't runtime requirements

If still over budget after above: remove the least-specific conventions first.
