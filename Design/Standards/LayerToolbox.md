# Review Support Toolbox

**Purpose**: Situational tools that help reviewers understand, challenge, and decide. Loaded by CreateDesign Step 4 and ValidateOutput Gate 2.

Activate 0-2 tools per layer based on context from CreateDesign Steps 2-3.

---

## Layer 1 — Grounding (Why trust this?)

| Tool | Activate When | How It Works |
|------|--------------|-------------|
| **Authority / expert citation** | Proposal contradicts intuition or team has low domain familiarity | Reference recognized experts or standards that validate the approach |
| **Industry practice** | Approach is novel for team but established elsewhere | "Company X does this" or "This is standard practice in..." |
| **Benchmarks / data** | Performance claims need proof; audience is data-driven | Concrete numbers, comparison tables, measured results |

## Layer 2 — Structure (How to review this?)

| Tool | Activate When | How It Works |
|------|--------------|-------------|
| **Likely objections** | You can predict top 2-3 objections; design is controversial | Address the objection where it affects criteria, assumptions, or option evaluation |
| **Incremental blocking** | Change is large; needs to feel manageable | Break into phases; each phase is a self-contained improvement |
| **Flexible surfaces** | Reviewers can improve the design in bounded areas | Mark 2-3 areas as open for input — must be genuinely negotiable |
| **Reducing goal distance** | Migration or multi-phase; progress needs to be visible | Show milestones, progress markers, intermediate deliverables |

## Layer 3 — Timing (Why decide now?)

| Tool | Activate When | How It Works |
|------|--------------|-------------|
| **Cost of delay** | Inaction has real, quantifiable cost; audience may default to "do nothing" | Show what happens if we do not act — in numbers, not abstractions |
| **Real deadline** | Genuine deadline or dependency creates time pressure | Reference the real constraint — never fabricate urgency |
| **Dependency constraint** | Prior decisions or dependencies make this the logical next step | "We already committed to X; this is the natural consequence" |
| **Prior decision continuity** | Proposal extends something already approved | "This builds on the approach we approved in [previous design]" |
| **Explicit decision request** | Full-scale docs where a clear decision is needed | State the exact decision request and what would change it |

---

## Scale Calibration

| Scale | Foundation | Layer 1 | Layer 2 | Layer 3 |
|-------|-----------|---------|---------|---------|
| Quick | All 6, light | 0-1 | 0-1 | 0 (just explicit request) |
| Standard | All 6, moderate | 1 | 1-2 | 0-1 |
| Full | All 6, thorough | 1-2 | 2-3 | 1-2 |
