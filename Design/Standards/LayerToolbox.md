# Layer Toolbox

**Purpose**: Situational persuasion tools organized by where they operate in the reviewer's cognitive journey. Loaded by CreateDesign Step 4 and ValidateOutput Gate 2.

Activate 0-2 tools per layer based on context from CreateDesign Steps 2-3.

---

## Layer 1 — Credibility (Why trust this?)

| Tool | Activate When | How It Works |
|------|--------------|-------------|
| **Authority / expert citation** | Proposal contradicts intuition or team has low domain familiarity | Reference recognized experts or standards that validate the approach |
| **Social proof (industry practice)** | Approach is novel for team but established elsewhere | "Company X does this" or "This is standard practice in..." |
| **Benchmarks / data** | Performance claims need proof; audience is data-driven | Concrete numbers, comparison tables, measured results |

## Layer 2 — Structure (How to comprehend this?)

| Tool | Activate When | How It Works |
|------|--------------|-------------|
| **Preemptive rebuttals** | You can predict top 2-3 objections; design is controversial | Address the objection before the reviewer raises it, with evidence |
| **Incremental blocking** | Change is large; needs to feel manageable | Break into phases; each phase is a self-contained improvement |
| **Concession surfaces** | Reviewers need ownership; design has genuinely flexible areas | Mark 2-3 areas as open for input — must be genuinely negotiable |
| **Reducing goal distance** | Migration or multi-phase; progress needs to be visible | Show milestones, progress markers, intermediate deliverables |

## Layer 3 — Momentum (Why say yes now?)

| Tool | Activate When | How It Works |
|------|--------------|-------------|
| **Penalty framing (cost of inaction)** | Inaction has real, quantifiable cost; audience may default to "do nothing" | Show what happens if we don't act — in numbers, not abstractions |
| **Scarcity / window of opportunity** | Genuine deadline or dependency creates time pressure | Reference the real constraint — never fabricate urgency |
| **Forcing mechanisms (existing commitments)** | Prior decisions or dependencies make this the logical next step | "We already committed to X; this is the natural consequence" |
| **Commitment/consistency** | Proposal extends something already approved | "This builds on the approach we approved in [previous design]" |
| **Verdict construction** | Full-scale docs where explicit approval is needed | Construct the exact decision request with the single strongest reason |

## Scale Calibration

| Scale | Foundation | Layer 1 | Layer 2 | Layer 3 |
|-------|-----------|---------|---------|---------|
| Quick | All 6, light | 0-1 | 0-1 | 0 (just explicit ask) |
| Standard | All 6, moderate | 1 | 1-2 | 0-1 |
| Full | All 6, thorough | 1-2 | 2-3 | 1-2 |
