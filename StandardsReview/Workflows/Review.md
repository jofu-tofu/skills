# StandardsReview Orchestrator

> You are a thin orchestrator. You spawn agents and check artifacts.
> You do NOT read workflow files yourself — agents read their own instructions.
> You do NOT write review findings — agents do that.

## Step 0: Resolve Paths

```bash
REPO_ROOT=$(git rev-parse --show-toplevel)
SKILL_ROOT=$REPO_ROOT/skills/StandardsReview
```

## Path Conventions

`$REVIEW_DIR` follows the PAI context system convention:
  `_output/contexts/[context-slug]/reviews/standardsreview/[YYYYMMDD-HHMMSS]`

The `[context-slug]` is auto-generated by the PAI context hook (UserPromptSubmit) from the user's prompt text — the orchestrator does not derive it. It appears in the hook's output as `Context folder: [path]`. Use this path directly.

If no context hook output is available, create one: `YYYYMMDD-HHMM-standardsreview-[branch-name]`.

## Step 1: Setup

Determine review mode from the user's request (diff vs audit). Then create the review directory:

```bash
mkdir -p $REVIEW_DIR
```

## Step 2: Gather Context

Spawn agent:
- **prompt:** `"Read $SKILL_ROOT/Workflows/GatherContext.md and execute it. Review target: [commit range or target path from user]. Write your output to: $REVIEW_DIR/context.md. Return ONLY the absolute path to context.md when done."`
- **subagent_type:** `general-purpose`

**CHECK:** Read `$REVIEW_DIR/context.md` — must exist and be non-empty.
- If missing -> retry once with same prompt.
- If missing again -> output error to user: "GatherContext failed after 2 attempts. Review aborted."

## Step 3: Select Dimensions

Spawn agent:
- **prompt:** `"Read $SKILL_ROOT/Workflows/SelectDimensions.md and execute it. Context file: $REVIEW_DIR/context.md. Dimensions directory: $SKILL_ROOT/Dimensions/. Write your output to: $REVIEW_DIR/dimensions.json. Return ONLY the absolute path to dimensions.json when done."`
- **subagent_type:** `general-purpose`

**CHECK:** Read `$REVIEW_DIR/dimensions.json` — must exist and contain a JSON array of dimensions.
- If missing -> retry once. If missing again -> error to user.

## Step 4: Spawn Review Agents

Read `$REVIEW_DIR/dimensions.json` — this is the ONLY artifact the orchestrator reads besides agent output file paths.

For EACH dimension in the list, spawn an agent IN PARALLEL (`run_in_background: true`):
- **prompt:** `"You are a language standards reviewer. Your rules: [dimension.path] — read this file for every rule you must check. Review context: $REVIEW_DIR/context.md — read this file for what changed and why. Files to review: [dimension.files]. For each violation found, output: RULE_ID, severity (CRITICAL/HIGH/MEDIUM/LOW), file, line, description, recommendation. If reviewing a diff, only flag issues in changed lines. Write your findings to: $REVIEW_DIR/dimension-[dimension.id].md. Return ONLY the absolute path to the file you wrote."`
- **subagent_type:** `general-purpose`

**SYNCHRONIZATION:** Store each agent's task_id and dimension id. After all spawned:
1. Call `TaskOutput(task_id, block=true)` for each agent to collect the file path it returned
2. Per-agent check: verify the returned path exists and is non-empty. If missing, log a warning and optionally retry that specific agent
3. Verify: collected path count matches `dimensions.json` entry count
4. Build the list of agent output paths for downstream steps

**The orchestrator does NOT read agent output content.** It only collects file paths.

## Step 5: Verify Claims and Generate Report

Spawn agent:
- **prompt:** `"Read $SKILL_ROOT/Workflows/VerifyAndReport.md and execute it. Agent output files: [list of $REVIEW_DIR/dimension-*.md paths from Step 4]. Context: $REVIEW_DIR/context.md. Write verified findings to: $REVIEW_DIR/verified-findings.md. Write report to: $REVIEW_DIR/report.md. Return the report content AND the path to report.md."`
- **subagent_type:** `general-purpose`

**CHECK:** `$REVIEW_DIR/report.md` exists.

Output the report to the user.
